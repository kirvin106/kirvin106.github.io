Puertos de red TCP y UDP

1. ¿Qué son los puertos de red TCP y UDP?

Los puertos de red son valores numéricos (0–65535) que, junto con una dirección IP y un protocolo (TCP o UDP), identifican una aplicación o servicio en un host. TCP (Transmission Control Protocol) y UDP (User Datagram Protocol) son capas de transporte que usan estos puertos para diferenciar conexiones y flujos de datos.

2. Notas rápidas

  • Los puertos del rango 0–1023 se llaman puertos bien conocidos.
  • Los puertos 1024–49151 son registrados por IANA para servicios específicos.
  • Los puertos 49152–65535 son dinámicos/privados.

3. Tabla: protocolos y puertos habituales

Protocolo Transporte Puerto(es) por defecto Comentarios
HTTP TCP 80 Tráfico web no cifrado. (HTTP/3 usa QUIC sobre UDP).
HTTPS TCP (TLS) 443 HTTP sobre TLS/SSL — tráfico web cifrado.
SMTP TCP 25 (relé), 587 (submission), 465 (smtps, histórico/secure) Envío de correo. 587 es el puerto moderno para submission con STARTTLS; 465 a veces usado para SMTP sobre TLS implícito.
POP3 TCP 110 (POP3), 995 (POP3S) Recuperación de correo (descarga); 995 es POP3 sobre TLS/SSL.
IMAP TCP 143 (IMAP), 993 (IMAPS) Acceso remoto a buzón; 993 es IMAP sobre TLS/SSL.
MIME — — MIME no es un protocolo de transporte con puerto: es un formato/extensión para correo (tipos de contenido).
FTP TCP 21 (control), 20 (data, modo activo) FTP usa dos conexiones: control (21) y datos (20 en modo activo). En modo pasivo el servidor abre puertos altos para la transferencia.
FTPS TCP (TLS) 990 (implícito), 21 (explícito + TLS) FTP con TLS/SSL. Puede usarse en modo implícito (990) o explícito via STARTTLS sobre 21.
SFTP TCP (sobre SSH) 22 SFTP es FTP sobre SSH (protocolo distinto); usa el puerto SSH por defecto.
BitTorrent TCP/UDP 6881–6889 (rango común), trackers: 6969; otros puertos dinámicos Cliente P2P; suele usar rangos registrados o puertos configurables por el usuario.
FastTrack (KaZaA, etc.) TCP/UDP 1214 (histórico / registrado) Protocolo P2P usado por KaZaA / Morpheus; versiones modernas pueden usar puertos variables.
IRC TCP 6667 (habitual), 6660–6669, 6697 (TLS) Chat en tiempo real; 6697 común para conexiones TLS/SSL.
XMPP (Jabber) TCP 5222 (cliente), 5269 (servidor) 5222 es el puerto para clientes (con STARTTLS); 5223 era uso histórico para SSL directo.
MTProto (Telegram) TCP/UDP 443, 80, 5222, 4244 (usos comunes / variaciones) MTProto no tiene un único puerto "oficial"; Telegram usa varios puertos y técnicas para atravesar NAT/firewalls.

4. Consejos prácticos

  • Siempre consulta el registro IANA para asignaciones oficiales de puertos.
  • Muchos servicios aceptan ser configurados para usar puertos distintos al por defecto — esto no es garantía de seguridad, solo obscurecimiento.
  • Si configuras un firewall, abre solo los puertos estrictamente necesarios y usa TLS siempre que sea posible.
